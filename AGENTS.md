## Architecture

- **SPA entry:** `src/main.tsx` ‚Üí `src/App.tsx` (BrowserRouter wrapping Routes).
- **Public layout sections** (order, rendered inline in App.tsx):
  TopBar ‚Üí Navbar ‚Üí Hero ‚Üí About ‚Üí CarRentals ‚Üí FAQ ‚Üí Footer (with Google Reviews panel) ‚Üí PwaInstallBanner
- **Admin routes** at `/admin` ‚Äî login, dashboard, unified cars page (Voitures + Tarifs + Transport + Configuration tabs), reservations list.
- **Path alias:** `@/*` ‚Üí `src/*` (configured in both tsconfig.json and vite.config.ts).
- **GSAP + ScrollTrigger** for scroll animations (`src/hooks/useScrollAnimation.ts`).
- **shadcn/ui** new-york style, lucide icons, `cn()` at `@/lib/utils`.
- **Remaining shadcn components:** `calendar.tsx`, `popover.tsx`, `select.tsx` only. All others cleaned out.

## i18n & Currency

- **Primary language: French** (default, fallbackLng: `'fr'`). English secondary. i18next via `src/i18n.ts`.
- **Currency context** at `src/lib/currency.tsx` ‚Äî supports EUR (default), USD, MAD. Conversion rates hardcoded inside the file.

## Season Logic

Haute saison = **July 1 ‚Üí August 25** (configurable via admin `/admin/cars` ‚Üí Configuration tab, stored in `settings` table). Outside that range is "normal" season. This affects pricing from the tariffs table.

## Supabase (Backend)

- **Client:** `src/lib/supabase.ts` ‚Äî Supabase JS SDK, initialized from `VITE_SUPABASE_URL` and `VITE_SUPABASE_ANON_KEY`. Types exported: `Car`, `Tariff`, `Reservation`, `TransportPrice`, `Setting`, `Franchise`.
- **Tables:** `cars`, `tariffs`, `reservations`, `admin_profiles`, `transport_prices`, `settings`, `franchises`. Full schema in `plan-supabase.md` and `scripts/migration-2026.sql`.
- **Admin auth:** Supabase Auth (email/password). Login at `/admin/login`. Protected by `ProtectedRoute.tsx`.
- **Seed:** `node scripts/seed.mjs` (requires `SUPABASE_SERVICE_ROLE_KEY` in `.env`). Creates admin user (`admin@yacouttours.com` / `Admin123!`), 10 cars from `src/data/cars.json` (now removed from public/), 16 tariff brackets, transport prices, settings, and franchises.
- **Migration status:** Fully migrated to Supabase. Old pipeline files cleaned up. WhatsApp booking still opens in parallel.

## Deployment

- **GitHub Pages** via `.github/workflows/pages.yml` (pushes `dist/` to `gh-pages` branch, runs on push to `master`).
- **Vercel** also supported (`vercel.json` SPA rewrites).
- Base URL: `/` locally, `/RENT/` when `GITHUB_ACTIONS` env is set.
- **Use `img(path)` from `@/lib/utils`** for asset paths ‚Äî it resolves the BASE_URL for GH Pages.

## PWA

- **VitePWA** plugin in `vite.config.ts` ‚Äî generates production service worker. In dev, service worker is unregistered by `src/main.tsx:7-16`.
- Fallback: `registerSW.js` / `sw.js` at root (not generated by VitePWA in dev).
- Workbox runtime caching: Google Fonts (1yr cache), images (30d cache, max 3MB per file).

## Toolchain Quirks

- **`components.json`** now has `"tailwind.config"` pointing to `"tailwind.config.js"` (was `"postcss.config.js"` ‚Äî shadcn bug fixed).
- **`verbatimModuleSyntax: true** in tsconfig ‚Äî use `import type` for type-only imports.
- **`noUnusedLocals`/`noUnusedParameters`** are on (tsconfig.app.json).
- **`erasableSyntaxOnly: true`** in tsconfig ‚Äî prohibits `enum`, `namespace`, `parameter properties`.
- **`next-themes`** and other unused packages removed from dependencies.
- **`kimi-plugin-inspect-react`** devDependency ‚Äî a React attribute inspector plugin (not a standard Vite plugin).
- ESLint uses the new flat config format (`eslint.config.js`), not `.eslintrc`.
